# Manual LilyGO YAML Setup

Most users should use the
[LilyGO YAML helper](tools/lilygo-yaml-helper.html). This page is for manual
setup or troubleshooting when you want to edit the ESPHome YAML yourself.

## Manual Overlay

The manual overlay is here:
[lilygo_cc1101_overlay.yaml](../esphome/examples/lilygo_cc1101_overlay.yaml).

Do not copy the overlay over a blank YAML file. Start with the YAML generated by
ESPHome Builder so your local Wi-Fi, API encryption key, OTA password, and
device identity remain intact.

## What To Merge

After ESPHome Builder creates the device YAML:

1. Keep the generated `esphome`, `wifi`, `api`, `ota`, and `logger` sections.
2. Ensure there is only one top-level `substitutions` section.
3. Add `proflame2_package_ref: "main"` to `substitutions`.
4. Ensure `esp32.framework.type` is `arduino`.
5. Add the Proflame2 package entries from the overlay to the existing
   `packages` section, or create that section if it does not exist.
6. Add the restart switch from the overlay only if your YAML does not already
   have a `platform: restart` switch.

The current LilyGO Proflame2 firmware is validated on the Arduino framework.
ESPHome Builder may create the starter YAML with `esp-idf`; change only the
framework type unless you know the board value is wrong.

## Package Reference

The `proflame2_package_ref` value tells ESPHome which version of the LilyGO
firmware configuration to download from GitHub.

The default is:

```yaml
proflame2_package_ref: "main"
```

This uses the latest repository version when you rebuild the LilyGO firmware.

Advanced users may pin this to a release tag such as `"v0.3.0"` for
reproducible firmware builds. If you pin a release tag, update it whenever you
upgrade the Proflame2 integration and rebuild the LilyGO firmware.
